1. What is Git?

Git is a distributed version control system that tracks changes in your code.

2. Installing Git

  • Windows: Download installer
  • MacOS: brew install git
  • Linux: sudo apt install git

3. Git Configuration

Set your user name and email:

  • git config --global user.name "Your Name"
  • git config --global user.email "email@example.com"

4. Creating a Repository

Initialize a new repository:

  • git init
  • Start tracking files

5. Cloning a Repository

Clone an existing repo:

  • git clone [url]

6. Git Status

Check the status of files:

  • git status

7. Staging Changes

Add files to staging area:

  • git add [file]
  • git add . for all files

8. Committing Changes

Save staged changes:

  • git commit -m "Message"

9. Viewing History

  • git log
  • git log --oneline

10. Branching

Work on features separately:

  • git branch
  • git checkout -b new-branch

11. Merging Branches

Combine changes from one branch to another:

  • git merge branch-name

12. Deleting Branches

  • git branch -d branch-name

13. Undoing Changes

  • git checkout -- [file]
  • git reset [file]
  • git revert [commit]

14. Remote Repositories

  • git remote add origin [url]
  • git push -u origin main
  • git pull origin main

15. Forking a Repository

Fork projects on GitHub to contribute.

16. Pull Requests

Propose changes to a repository using PRs.

17. Tagging

  • git tag v1.0
  • git push origin --tags

18. Stashing

  • git stash
  • git stash pop

19. Rebasing

  • git rebase branch-name

20. Collaboration Best Practices

Write clear commit messages, review PRs, and maintain a clean history.

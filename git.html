Master version control with Git for efficient code management and collaboration

Chapter 1: Git Basics

What is Git?

Git is a distributed version control system that helps developers track changes in their code, collaborate with others, and manage different versions of their projects efficiently.

Key Features

  • Distributed: Every developer has full project history
  • Branching & Merging: Easy to create and merge branches
  • Fast & Efficient: Optimized for performance
  • Open Source: Free to use and modify
  • Industry Standard: Used by most development teams

Installing Git

Install Git on your operating system:

Installation Commands

# Windows: Download from git-scm.com
# macOS: Using Homebrew
brew install git

# Linux (Ubuntu/Debian)
sudo apt update
sudo apt install git

# Verify installation
git --version

Basic Git Configuration

# Set your username and email
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

# Check your configuration
git config --list

# Set default branch name
git config --global init.defaultBranch main

Chapter 2: Getting Started

Creating Your First Repository

Initialize and set up a Git repository:

Repository Setup

# Create a new directory
mkdir my-project
cd my-project

# Initialize Git repository
git init

# Check repository status
git status

# Clone an existing repository
git clone https://github.com/username/repository.git
git clone https://github.com/username/repository.git new-folder-name

Basic Workflow

The fundamental Git workflow: add, commit, push

Basic Git Commands

# Create some files
echo "# My Project" > README.md
touch app.js index.html

# Check status
git status

# Add files to staging area
git add README.md
git add .                    # Add all files
git add *.js                 # Add all JS files

# Commit changes
git commit -m "Initial commit"
git commit -m "Add main application files"

# View commit history
git log
git log --oneline           # Compact view

Chapter 3: Branching & Merging

Working with Branches

Branches allow you to work on features independently:

Branch Operations

# List all branches
git branch                    # Local branches
git branch -r                 # Remote branches
git branch -a                 # All branches

# Create a new branch
git branch feature-login
git checkout -b feature-payment   # Create and switch

# Switch between branches
git checkout main
git checkout feature-login
git switch main               # Newer alternative

# Delete a branch
git branch -d feature-old      # Safe delete
git branch -D feature-old      # Force delete

Merging and Rebasing

Combine work from different branches:

Merge and Rebase

# Merge a branch into current branch
git checkout main
git merge feature-login

# Rebase current branch onto another
git checkout feature-login
git rebase main

# Interactive rebase (last 3 commits)
git rebase -i HEAD~3

# Resolve merge conflicts
# Edit conflicted files, then:
git add .
git rebase --continue
# OR for merge conflicts:
git add .
git commit -m "Merge feature branch"

Chapter 4: Remote Repositories

Working with Remote Repositories

Collaborate with others using remote repositories like GitHub, GitLab, or Bitbucket:

Remote Repository Operations

# Add a remote repository
git remote add origin https://github.com/username/repo.git

# View remote repositories
git remote -v

# Push to remote repository
git push -u origin main          # First time
git push origin main              # Subsequent pushes
git push origin feature-login     # Push specific branch

# Pull changes from remote
git pull origin main
git fetch origin                   # Download without merging
git merge origin/main             # Merge fetched changes

# Clone and work with remotes
git clone https://github.com/username/repo.git
cd repo
git remote -v                     # Check remotes

Collaboration Workflow

Standard workflow for team collaboration:

Team Collaboration

# Fork & Clone workflow (common on GitHub)
# 1. Fork repository on GitHub
# 2. Clone your fork
git clone https://github.com/your-username/repo.git

# 3. Add original repo as upstream
git remote add upstream https://github.com/original-owner/repo.git

# 4. Create feature branch
git checkout -b my-feature

# 5. Make changes and commit
git add .
git commit -m "Add new feature"

# 6. Push to your fork
git push origin my-feature

# 7. Create Pull Request on GitHub

# 8. Sync with upstream
git fetch upstream
git merge upstream/main

Chapter 5: Advanced Git

Undoing Changes

Various ways to undo changes in Git:

Undo Operations

# Undo unstaged changes
git checkout -- filename         # Single file
git checkout -- .                 # All files
git restore filename             # Newer alternative

# Unstage files
git reset filename               # Unstage single file
git reset                        # Unstage all files

# Amend last commit
git commit --amend -m "New message"
git add forgotten-file.js
git commit --amend --no-edit

# Reset to previous commit
git reset --soft HEAD~1           # Keep changes staged
git reset --mixed HEAD~1          # Keep changes unstaged (default)
git reset --hard HEAD~1           # Discard all changes

# Revert a commit (safe for shared history)
git revert HEAD                   # Revert last commit
git revert abc1234                # Revert specific commit

Stashing and Tagging

Temporary storage and version tagging:

Stash and Tags

# Stash current changes
git stash                         # Stash all changes
git stash push -m "Work in progress"
git stash save "Feature work"

# List and apply stashes
git stash list                    # Show all stashes
git stash apply                   # Apply latest stash
git stash apply stash@{1}         # Apply specific stash
git stash pop                     # Apply and remove
git stash drop stash@{1}          # Remove specific stash

# Working with tags
git tag v1.0.0                    # Create lightweight tag
git tag -a v1.0.0 -m "Version 1.0.0 release"
git tag                           # List all tags
git push origin v1.0.0            # Push tag to remote
git push origin --tags            # Push all tags

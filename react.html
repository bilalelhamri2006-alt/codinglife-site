Chapter 1: React Introduction

What is React?

React is a JavaScript library for building user interfaces, especially for web applications. It's component-based and uses a virtual DOM for better performance.

Why React?

  • Component-Based: Reusable UI components
  • Virtual DOM: Better performance
  • JSX: HTML-like syntax in JavaScript
  • Unidirectional Data Flow: Predictable state management
  • Large Ecosystem: Rich library support

Setting Up React

There are multiple ways to start a React project:

Create React App (Recommended)

# Create new React project
npx create-react-app my-app
cd my-app
npm start

Vite (Fast Alternative)

# Create project with Vite
npm create vite@latest my-react-app -- --template react
cd my-react-app
npm install
npm run dev

Chapter 2: JSX and Components

What is JSX?

JSX is a syntax extension for JavaScript that looks similar to HTML. It allows you to write HTML-like code in JavaScript.

JSX Examples

// Simple JSX
const element = <h1>Hello, World!</h1>;

// JSX with expressions
const name = "Sarah";
const greeting = <h2>Hello, {name}!</h2>;

// JSX with attributes
const image = <img src="photo.jpg" alt="Description" />;

Functional Components

Components are the building blocks of React applications:

Functional Component

// Simple functional component
function Welcome() {
  return (
    <div>
      <h1>Welcome to React!</h1>
      <p>This is my first component.</p>
    </div>
  );
}

// Arrow function component
const UserCard = () => {
  return (
    <div className="user-card">
      <h3>John Doe</h3>
      <p>Software Developer</p>
    </div>
  );
};

Chapter 3: Props and State

Props (Properties)

Props are used to pass data from parent to child components:

Using Props

// Component receiving props
function Greeting(props) {
  return <h1>Hello, {props.name}!</h1>;
}

// Using the component
<Greeting name="Alice" />

// Destructuring props
function UserInfo({ name, age, email }) {
  return (
    <div>
      <h3>{name}</h3>
      <p>Age: {age}</p>
      <p>Email: {email}</p>
    </div>
  );
}

State with useState Hook

State allows components to manage and update their data:

useState Hook

// Import useState
import React, { useState } from 'react';

function Counter() {
  // Declare state variable
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>You clicked {count} times</p>
      <button onClick={() => setCount(count + 1)}>
        Click me
      </button>
    </div>
  );
}

Chapter 4: Event Handling

Handling Events

React events are similar to DOM events but with some syntax differences:

Event Handling Examples

function ButtonComponent() {
  const handleClick = () => {
    alert('Button clicked!');
  };

  const handleInputChange = (event) => {
    console.log(event.target.value);
  };

  return (
    <div>
      <button onClick={handleClick}>
        Click Me
      </button>
      
      <input 
        type="text" 
        onChange={handleInputChange}
        placeholder="Type something..."
      />
    </div>
  );
}

Form Handling

Controlled components in React:

Controlled Form

function LoginForm() {
  const [formData, setFormData] = useState({
    username: '',
    password: ''
  });

  const handleChange = (e) => {
    const { name, value } = e.target;
    setFormData({
      ...formData,
      [name]: value
    });
  };

  const handleSubmit = (e) => {
    e.preventDefault();
    console.log('Form submitted:', formData);
  };

  return (
    <form onSubmit={handleSubmit}>
      <input
        type="text"
        name="username"
        value={formData.username}
        onChange={handleChange}
        placeholder="Username"
      />
      <input
        type="password"
        name="password"
        value={formData.password}
        onChange={handleChange}
        placeholder="Password"
      />
      <button type="submit">Login</button>
    </form>
  );
}

Chapter 5: useEffect and Lifecycle

useEffect Hook

The useEffect hook lets you perform side effects in functional components:

useEffect Examples

import React, { useState, useEffect } from 'react';

function UserProfile() {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(true);

  // useEffect with empty dependency array (runs once)
  useEffect(() => {
    // Simulate API call
    setTimeout(() => {
      setUser({ name: 'John Doe', email: 'john@example.com' });
      setLoading(false);
    }, 2000);
  }, []); // Empty array = run only on mount

  if (loading) {
    return <div>Loading...</div>;
  }

  return (
    <div>
      <h2>{user.name}</h2>
      <p>{user.email}</p>
    </div>
  );
}

Conditional Effects

You can run effects conditionally based on dependencies:

Effect with Dependencies

function SearchComponent() {
  const [query, setQuery] = useState('');
  const [results, setResults] = useState([]);

  // This effect runs whenever 'query' changes
  useEffect(() => {
    if (query.length > 2) {
      // Simulate search API call
      fetchResults(query).then(setResults);
    }
  }, [query]); // Re-run when query changes

  return (
    <div>
      <input
        value={query}
        onChange={(e) => setQuery(e.target.value)}
        placeholder="Search..."
      />
      <ul>
        {results.map(result => (
          <li key={result.id}>{result.name}</li>
        ))}
      </ul>
    </div>
  );
}
